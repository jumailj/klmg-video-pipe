# Quick Reference Card

## Connection Flow (Fixed)

```
BEFORE (Broken):
Streamer connects → Hub updates → OBS can't trigger offer
(Offer never sent unless OBS was already connected)

AFTER (Fixed):
OBS clicks "Select Streamer"
  ↓
Hub receives {type: "select-streamer", targetStreamerId: "xxx"}
  ↓
Hub routes to matching Streamer
  ↓
Streamer receives message
  ↓
Streamer calls makeOffer()
  ↓
Offer sent to OBS
  ↓
OBS receives offer & sends answer
  ↓
✅ WebRTC connection established
```

```
❌ Problems:
"bind: Only one usage" → Port 8080 in use, kill process
relay: streamer xxx not found → Wrong room ID
relay: no OBS viewers in room → OBS hasn't connected yet
```

## If It's Still Not Working

1. **Check server is actually running**
   - Look for: "VOD dashboard listening on :8080"
   - If not, port 8080 in use → kill vodapp process

2. **Check WebSocket connects**
   - Browser F12 → Network tab → WS
   - Should see: wss://localhost:8080/ws?room=...

3. **Check message flow**
   - Streamer console: Look for "OBS selecting this streamer"
   - OBS console: Look for received messages
   - Server logs: Look for relay messages

4. **Check network is accessible**
   - Can ping: localhost:8080 (curl http://localhost:8080/)
   - With ngrok: is tunnel still active? (no timeout)

5. **Last resort**
   - Kill process: `Get-Process vodapp | Stop-Process -Force`
   - Rebuild: `go build`
   - Restart: `.\vodapp.exe`
   - Clear browser cache (Ctrl+Shift+Delete)
